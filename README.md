## Agents that build the way you would

AgentOS helps you shape better specs, keeps agents aligned with your standards, and adapts workflows to any task — all in a lightweight system that fits how you already build.

Works alongside Claude Code, Cursor, Codex, and other AI tools. Any language, any framework.

**Core capabilities:**

- **Discover Standards** — Extract patterns and conventions from your codebase into documented standards
- **Deploy Standards** — Intelligently inject relevant standards based on what you're building
- **Shape Spec** — Create better plans that lead to better builds with structured specifications
- **Self-Improve** — Ralph Loop: iterate, learn, and improve through continuous feedback cycles

### What's Inside

AgentOS combines three proven systems into one unified framework:

1. **Ralph Loop** — Iterative self-improvement through continuous feedback cycles
2. **Superpowers** — Iron-law enforcement, hard gates, and anti-slop patterns
3. **Workflow Engine** — Dynamic task decomposition and capability composition

The result: **any AI coding agent** can be upgraded to operate at the level of the most advanced internal systems.

### Quick Start

```bash
# Clone to home directory
git clone https://github.com/your-org/agent-os.git ~/agent-os

# Install into your project
cd /path/to/your/project
~/agent-os/scripts/install-base.sh
```

See [docs/INSTALLATION.md](docs/INSTALLATION.md) for full installation guide.

### Project Status

**v0.1.0** — Alpha. The workflow engine runs end-to-end (classify → compose → execute). Tests pass (44/44). Installation scripts work on Unix and Windows.

**What's implemented:**
- Task classifier (8 task types, 10 domains, 3 complexity levels)
- Capability registry (71 capabilities cataloged)
- Workflow composer (7 templates with adaptation)
- Execution engine (generates phase-by-phase instructions)
- Adaptation layer (capability substitution, error rate tracking, parallelism adjustment)
- Session management, memory extraction, context deduplication
- Cross-platform install/uninstall scripts
- Profile system with inheritance chains

**What's planned:**
- Actual code execution (currently generates instructions for agents to follow)
- Ralph Loop as an automated cycle (currently a conceptual pattern)
- GitHub integration (architecture documented, not implemented)
- Unknown task handling framework (architecture documented, not implemented)
- MCP tool execution (manager implemented, execution is placeholder)
