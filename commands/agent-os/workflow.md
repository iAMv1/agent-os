# /workflow — Compose and execute workflow

## Description
Analyze the current task and compose an optimal workflow using available capabilities.

## Usage
Run this command when you want to:
- Start a complex task that needs structured handling
- Let the system determine the best approach
- Execute with parallel/sequential phases

## Available Workflows
- **Requirements Gathering** — For understanding what to build
- **Architecture & Design** — For planning system structure
- **Implementation** — For building features
- **Testing & QA** — For quality verification
- **Deployment & DevOps** — For shipping code
- **Maintenance & Monitoring** — For ongoing support
- **Full SDLC** — For end-to-end project work

## Process
1. Classify the task (type, complexity, domain)
2. Compose optimal workflow from capabilities
3. Execute phases with parallel/sequential handling
4. Adapt based on execution feedback

## Output
- Composed workflow with phases
- Execution results per phase
- Adaptation log
